## 一、产品概述

### 1.1 产品定位
HerGallery 是一个女性主题的链上创作与策展平台，让任何人都可以发起关于女性的主题展厅，通过投稿二创作品和社区推荐机制，共同构建永久存证、不可篡改的女性故事档案库。

### 1.2 核心价值
| 价值 | 说明 |
|------|------|
| 永久存证 | 所有内容上链，不可删除、不可篡改 |
| 无需许可 | 任何人可发起展厅、投稿，无中心化审核 |
| 社区推荐 | 推荐数决定内容排序，优质内容自然浮现 |
| 创作激励 | POAP事件机制为未来激励奠定基础 |

### 1.3 目标用户
| 用户类型 | 使用场景 |
|----------|----------|
| 策展人 | 为女性事件/人物发起展厅，组织内容征集 |
| 创作者 | 向展厅投稿二创作品、证言、截图 |
| 推荐者 | 推荐喜欢的作品，帮助优质内容脱颖而出 |
| 观众 | 浏览展厅内容，了解女性议题 |

### 2.2 展厅模块

#### 2.2.1 创建展厅

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 标题 | string | 是 | 展厅名称，如“云吃吃事件”，最长50字符 |
| 描述 | string | 是 | 展厅背景介绍，最长500字符 |
| 封面图 | file | 是 | 图片上传至IPFS，支持jpg/png，最大2MB |

**业务规则**：
- 创建需质押 **0.001 AVAX**
- 质押金额在展厅关闭后可退还（MVP不做关闭功能，但合约已预留）
- 创建成功后，用户成为该展厅的“策展人”
- 展厅ID自动递增，从1开始

**交互流程**：
1. 用户点击“发起新展厅”按钮
2. 弹出表单，填写标题、描述、上传封面
3. 点击“确认创建”，钱包弹出交易确认
4. 等待链上确认，成功后跳转到展厅详情页

**验收标准**：
- [ ] 表单验证：标题和描述不能为空
- [ ] 封面上传成功后显示预览
- [ ] 交易确认后页面跳转
- [ ] 新展厅出现在首页列表顶部

### 2.3 投稿模块

#### 2.3.1 提交投稿

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 类型 | 下拉选择 | 是 | 二创 / 证言 / 截图 / 链接 |
| 内容 | 文件/文本 | 是 | 根据类型不同，上传图片或填写文本 |
| 标题 | string | 是 | 投稿标题，最长50字符 |
| 说明 | string | 是 | 创作说明，最长200字符 |

**各类型处理规则**：

| 类型 | 内容输入方式 | 存储方式 |
|------|--------------|----------|
| 二创 | 图片上传 / 文本输入 | 图片存IPFS，文本存链上 |
| 证言 | 文本输入 | 链上直接存储 |
| 截图 | 图片上传 | IPFS存储，链上存哈希 |
| 链接 | URL输入 | 链上存储 |

**业务规则**：
- 每人可无限次投稿（无限制）
- 投稿成功后，投稿显示在列表末尾（等待推荐）
- 如果是该用户的首次投稿，合约触发 `FirstSubmission` 事件

**交互流程**：
1. 在展厅详情页点击“我要投稿”
2. 选择类型，根据类型展示不同输入框
3. 填写标题、说明，上传/输入内容
4. 点击“提交”，钱包弹出交易确认
5. 等待确认，成功后刷新投稿列表

**验收标准**：
- [ ] 类型切换时输入框正确变化
- [ ] 图片上传成功后显示预览
- [ ] 文本投稿有字数限制提示
- [ ] 交易确认后列表自动刷新

### 2.4 推荐模块

#### 2.4.1 推荐功能

| 维度 | 说明 |
|------|------|
| 操作 | 用户点击 ❤️ 按钮 |

- [ ] 推荐成功后按钮变灰
- [ ] 同一用户不能重复推荐同一投稿
- [ ] 推荐数达到10时控制台可见事件日志

#### 3.1.1 数据结构

```solidity
struct Exhibition {
    uint256 id;
    string title;
    string contentHash;      // IPFS哈希 - 主题介绍
    string coverHash;        // IPFS哈希 - 封面图
    address curator;
    uint256 createdAt;
    bool isActive;
    uint256 submissionCount;
}

struct Submission {
    uint256 id;
    uint256 exhibitionId;
    string contentType;  // art / testimony / screenshot / link
    string contentHash;
    string title;
    string description;
    address creator;
    uint256 recommendCount;
    uint256 createdAt;
}
```

#### 3.1.2 核心函数

| 函数 | 参数 | 说明 |
|------|------|------|
| createExhibition | title, contentHash, coverHash | 创建展厅，需质押，contentHash为主题介绍IPFS哈希 |
| submitToExhibition | exhibitionId, contentType, contentHash, title, description | 投稿，contentHash为内容IPFS哈希 |
| recommend | exhibitionId, submissionId | 推荐投稿 |
| getSubmissions | exhibitionId | 获取所有投稿 |
| setUsername | username | 设置用户昵称（1-20字符，仅限一次） |
| getRecommendedStatus | exhibitionId, user | 批量查询用户对某展厅所有投稿的推荐状态 |

#### 3.1.3 事件

| 事件 | 参数 |
|------|------|
| ExhibitionCreated | id, title, curator |
| SubmissionCreated | id, exhibitionId, creator |
| Recommended | submissionId, recommender, newCount |
| FirstSubmission | user, submissionId |
| RecommendMilestone | creator, submissionId, recommendCount |

#### 2.2.1 创建展厅（更新）

### 2.3 投稿模块（更新）

#### 2.3.1 提交投稿（统一IPFS存储）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 类型 | 下拉选择 | 是 | 二创 / 证言 / 截图 / 链接 |
| 内容 | 文件/文本 | 是 | 根据类型不同，上传图片或填写文本 |
| 标题 | string | 是 | 投稿标题，最长50字符 |
| 说明 | string | 是 | 创作说明，最长200字符 |

**统一存储方案**：
- **所有类型的内容统一打包上传至IPFS**
- 链上仅存储IPFS哈希（contentHash）
- 大幅降低Gas成本（从多次存储变为一次存储）

**各类型内容打包规则**：

| 类型 | 内容输入方式 | 打包数据结构 |
|------|--------------|--------------|
| 二创 | 图片上传 / 文本输入 | `{ type: "art", imageHash: "xxx", text: "xxx" }` |
| 证言 | 文本输入 | `{ type: "testimony", text: "xxx" }` |
| 截图 | 图片上传 | `{ type: "screenshot", imageHash: "xxx" }` |
| 链接 | URL输入 | `{ type: "link", url: "xxx" }` |

**业务规则更新**：
- 每人可无限次投稿（无限制）
- 投稿成功后，投稿显示在列表末尾（等待推荐）
- 如果是该用户的首次投稿，合约触发 `FirstSubmission` 事件
- **Gas成本估算：约50,000-80,000 Gas（仅存储哈希）**

**交互流程更新**：
1. 在展厅详情页点击“我要投稿”
2. 选择类型，根据类型展示不同输入框
3. 填写标题、说明，上传/输入内容
4. 点击“提交”，**前端将内容打包成JSON上传至IPFS**，获得哈希
5. 钱包弹出交易确认（传递哈希）
6. 等待确认，成功后刷新投稿列表

**验收标准更新**：
- [ ] 类型切换时输入框正确变化
- [ ] 图片上传成功后显示预览
- [ ] 文本投稿有字数限制提示
- [ ] **内容正确打包上传IPFS**
- [ ] 交易确认后列表自动刷新
